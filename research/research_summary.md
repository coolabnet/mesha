# Research Summary: Automating LibreMesh Testing with Virtual QEMU Nodes

## Key Resources Identified

### 1. Official LibreMesh Virtualization Documentation

- **URL**: <https://github.com/libremesh/lime-packages/blob/master/VIRTUALIZING.md>
- **Content**: Comprehensive guide to running LibreMesh in QEMU
- **Key Points**:
  - Requires `generic-rootfs.tar.gz` and `generic-kernel.bin` files
  - Generated by compiling LibreMesh with x86 target and ramfs.bzImage option
  - LibreRouterOS distributes these images in releases
  - Limitations: ICMPv4 doesn't work between QEMU nodes (but ICMPv6/ping6 works)
  - Most functionality works as expected: initialization scripts, lime-config, ubus/rpcd, lime-app

### 2. GSoC 2025 Project: Adding Wi-Fi Support to QEMU Simulations

- **URL**: <https://blog.freifunk.net/2025/09/01/gsoc-2025-bringing-wi-fi-support-to-qemu-simulations-for-libremesh/>
- **Content**: Final report of GSoC 2025 project for Wi-Fi support in LibreMesh QEMU simulation
- **Key Points**:
  - Uses mac80211_hwsim kernel module for Wi-Fi simulation
  - Integrated with existing QEMU setup to enable virtual Wi-Fi
  - Features kmod-mac80211-hwsim package for OpenWrt
  - Demonstrates AP-STA setup within virtual machine
  - Uses vwifi project to connect multiple VMs through Wi-Fi interfaces
  - Addresses limitations of Wi-Fi bands and 802.11ax restrictions

### 3. GitHub Issue: Adding Wi-Fi Support to QEMU Simulations (#1178)

- **URL**: <https://github.com/libremesh/lime-packages/issues/1178>
- **Content**: Main issue tracking Wi-Fi support for QEMU simulations
- **Sub-issues**:
  - Review existing tools (mac80211_hwsim, vwifi, OpenWrt test frameworks)
  - QEMU LibreMesh instance with WiFi capabilities
  - Create mesh network with wifi simulated QEMU instances
  - Develop scripts using rpcd interfaces for network diagnostics
  - Script Development for Network Testing (automation)
  - Integration with Existing LibreMesh Testing Framework
  - Documentation and Final Testing

### 4. Alternative Virtualization Guide

- **URL**: <https://hackmd.io/@fd-FiKStTBGvsTvI8OKjhA/BJJuUmHMs>
- **Content**: Step-by-step guide for running LibreMesh in virtual machines
- **Key Points**:
  - Clone lime-packages repository
  - Install dnsmasq, iptables, qemu-system-x86
  - Run ./tools/qemu_dev_start script
  - Detailed instructions for single and multi-node setups
  - Internet access configuration guidance

### 5. LibreMesh SDK Information

- **URL**: <https://libremesh.org/build/lime-sdk.html>
- **Content**: Information about LibreMesh SDK and build system
- **Key Points**:
  - Uses OpenWRT SDK and ImageBuilder
  - Cooker tool for generating packages and firmware
  - Dependencies for building in running system (Debian/Ubuntu)
  - Docker container build option

### 6. Google Summer of Code Project Details

- **URL**: <https://summerofcode.withgoogle.com/programs/2025/projects/yBmMMWDN>
- **Content**: Official GSoC 2025 project page
- **Key Benefits**:
  - Improved testing framework for "almost real" tests
  - Enhanced automation without physical routers
  - Scripted tests adaptable to real hardware
  - Collaboration opportunities with OpenWrt
  - Lower barrier to entry for new contributors

### 7. QEMU Continuous Integration Documentation

- **URL**: <https://www.qemu.org/docs/master/devel/testing/ci.html>
- **Content**: QEMU's CI practices
- **Key Points**:
  - Automated tests: unit, functional, and other tests
  - Gating tests before merging pull requests
  - CI environment variables for customization
  - Maintainer-controlled job variables

## Strategy for Automated Testing

Based on the research, here's a comprehensive strategy for automating LibreMesh testing with virtual LibreMesh nodes using QEMU:

### Phase 1: Environment Setup

1. **Prerequisites Installation**:
   - Install QEMU system components (`qemu-system-x86`)
   - Install networking tools (`dnsmasq`, `iptables`)
   - Install build dependencies for LibreMesh compilation
   - Install kernel module dependencies for `mac80211_hwsim`

2. **Environment Preparation**:
   - Clone LibreMesh repositories (lime-packages, potentially others)
   - Set up build environment for generating QEMU-compatible images
   - Configure networking for inter-VM communication

### Phase 2: Virtual Node Creation

1. **Image Generation**:
   - Build LibreMesh firmware targeting x86 architecture
   - Generate `generic-rootfs.tar.gz` and `generic-kernel.bin`
   - Alternatively, use pre-built images from LibreRouterOS releases

2. **Single Node Validation**:
   - Test basic QEMU launch with generated images
   - Verify basic functionality (initialization scripts, services)
   - Confirm network connectivity (noting ICMPv4 limitation)

### Phase 3: Wireless Network Simulation

1. **Wi-Fi Capability Integration**:
   - Incorporate `kmod-mac80211-hwsim` into LibreMesh build
   - Patch any Wi-Fi band/configuration issues as identified in GSoC work
   - Validate Wi-Fi functionality within single VM

2. **Multi-Node Wireless Network**:
   - Implement vwifi or similar solution for inter-VM Wi-Fi communication
   - Create scalable topology for multiple virtual nodes
   - Test mesh formation and protocol functionality

### Phase 4: Automation Framework

1. **Script Development**:
   - Create scripts for automated VM lifecycle management
   - Develop network configuration scripts for various test scenarios
   - Implement rpcd-based diagnostics for remote testing

2. **Test Suite Design**:
   - Define test cases for core LibreMesh functionality
   - Create parameterized tests for different network configurations
   - Implement both unit and integration tests

### Phase 5: CI/CD Integration

1. **Automation Pipeline**:
   - Integrate with existing CI systems (GitHub Actions, GitLab CI)
   - Set up automated testing on pull requests and commits
   - Implement gating checks for code quality

2. **Reporting and Metrics**:
   - Establish test result reporting mechanisms
   - Create dashboards for test coverage and trends
   - Implement failure analysis and debugging aids

### Phase 6: Validation and Documentation

1. **Cross-Validation**:
   - Compare virtual test results with physical hardware tests
   - Validate automation scripts against manual procedures
   - Ensure compatibility with existing testing frameworks

2. **Documentation**:
   - Document setup procedures and usage guidelines
   - Create troubleshooting guides for common issues
   - Maintain updated examples and best practices
